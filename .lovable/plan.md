
מובן. בדקתי את הקוד של עמוד חשבונית מס קבלה, והבעיה האמיתית היא לא רק בשורת הסיכום אלא בלוגיקת החישוב עצמה.

מה קורה עכשיו:
- שדה `items.[i].total` מחושב תמיד כ־`quantity * unitPrice - discount`, בלי קשר לכפתור המע"מ.
- אחר כך בסיכום, אם `includeVat` דלוק, הקוד מתייחס למחיר כאילו הוא כבר כולל מע"מ ומחלץ ממנו את המע"מ.
- זה הפוך ממה שביקשת.

מה אתה רוצה בפועל:
- כשהכפתור כבוי:
  - המחיר שהוזן נשאר כמו שהוא
  - סה"כ מע"מ = 0
  - סה"כ הפריט = המחיר שהוזן כפול כמות פחות הנחה
- כשהכפתור דלוק:
  - המערכת מוסיפה מע"מ אוטומטית
  - סה"כ הפריט משתנה אוטומטית
  - התצוגה המקדימה מתעדכנת מיד
  - וגם המסמך/PDF יוצא עם אותם מספרים

תוכנית תיקון לעמוד `TaxInvoiceReceipt.tsx`:
1. לאחד את משמעות המחיר:
   - `unitPrice` יהיה מחיר בסיס לפני מע"מ.
   - זה יהיה עקבי בכל הטופס, בתצוגה המקדימה ובמסמך.

2. לתקן את חישוב שורת הפריט ב־`useEffect`:
   - היום: `total = subtotal - discount`
   - צריך להיות:
     - בסיס: `base = quantity * unitPrice - discount`
     - אם `includeVat = false` => `total = base`
     - אם `includeVat = true` => `total = base + vat`
   - כך המשתמש יראה מיד שהסה"כ של הפריט משתנה כשהטוגל נדלק.

3. לתקן את `calculateFinancialSummary`:
   - לא לחלץ מע"מ מתוך `item.total`.
   - לחשב לפי מחיר הבסיס של כל פריט:
     - `base = quantity * unitPrice - discount`
     - אם הטוגל כבוי: להוסיף לבסיס בלבד, מע"מ 0
     - אם הטוגל דלוק: לחשב `vat = base * rate/100`
   - הסיכום יציג תמיד:
     - נטו
     - מע"מ
     - סה"כ כולל

4. לוודא שה־preview הימני והכרטיס במובייל משתמשים באותו `financialSummary`:
   - כך כשמכבים מע"מ יופיע `₪0.00`
   - וכשמדליקים, המע"מ והסה"כ יזוזו מיידית

5. לוודא שהפקת המסמך משתמשת בדיוק באותם נתונים:
   - `onSubmit`
   - `handleDownloadPDF`
   - `generateTaxInvoiceReceiptPDF`
   כך שהמסמך הסופי יהיה זהה למה שרואים לפני ההפקה.

6. לעדכן גם את טקסט המע"מ בתוך טבלת הפריטים ב־PDF:
   - כשהכפתור כבוי לא להציג `+ 18%` כאילו מתווסף אחר כך, אם בפועל סה"כ הפריט כבר מוצג ללא מע"מ
   - להציג ניסוח ברור בהתאם למצב הטוגל כדי לא לבלבל

למה זה יפתור את הבעיה:
- כרגע יש נתק בין "מחיר הפריט", "סיכום המע"מ" ו"המסמך הסופי".
- אחרי התיקון, כל המערכת תעבוד מאותו כלל אחד:
  - כבוי = בלי מע"מ בכלל
  - דלוק = מוסיפים מע"מ אוטומטית גם לפריט וגם לסיכום

פרטים טכניים:
```text
base = quantity * unitPrice - discount

if includeVat OFF:
  item.total = base
  vat = 0

if includeVat ON:
  vat = base * (vatRate / 100)
  item.total = base + vat
```

קבצים רלוונטיים:
- `src/pages/TaxInvoiceReceipt.tsx`
- `src/utils/pdf/tax-invoice-receipt-pdf.ts`

אחרי זה נוכל לעבור עמוד עמוד בדיוק באותה שיטה, אבל קודם נכון לסגור את חשבונית מס קבלה בצורה נקייה ועקבית.
